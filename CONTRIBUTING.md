# Contributing to Nova Framework

Thank you for your interest in Nova Framework!

## How to contribute

- ✅ **Issues** — bug reports, questions, and feature suggestions are welcome.
- ✅ **Pull Requests** — contributions are welcome. Fork the repository, create
  a feature branch, and open a Pull Request against the default branch. For
  large or breaking changes, please open an Issue first so we can discuss the
  approach before you invest significant effort.

## Reporting issues

- Search existing issues first to avoid duplicates.
- Use the issue templates (Bug report / Feature request).
- Include the Nova version, Unity version, target platform, reproduction steps,
  and relevant logs.

## Security

Do **not** report security vulnerabilities through public issues. See
[SECURITY.md](./SECURITY.md).

## Code of Conduct

By participating in this project you agree to abide by our
[Code of Conduct](./CODE_OF_CONDUCT.md).

# 为 Nova Framework 贡献

感谢你对 Nova Framework 的关注！

## 如何贡献

- ✅ **Issue** —— 欢迎 bug 反馈、问题咨询、功能建议。
- ✅ **Pull Request** —— 欢迎贡献代码。请 fork 仓库、新建特性分支，向默认分支提交
  Pull Request。对于较大或破坏性的改动，请先开 Issue 讨论方案，再投入较多精力。

## 提交 Issue

- 先搜索已有 issue，避免重复。
- 使用 issue 模板（Bug 反馈 / 功能建议）。
- 提供 Nova 版本、Unity 版本、目标平台、复现步骤与相关日志。

## 安全问题

**请勿**通过公开 issue 报告安全漏洞，详见 [SECURITY.md](./SECURITY.md)。

## 行为准则

参与本项目即表示你同意遵守我们的 [行为准则](./CODE_OF_CONDUCT.md)。
